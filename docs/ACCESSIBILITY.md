# ACCESSIBILITY — WCAG 2.2 AA Report

Owner: `flutter-engineer`. Regenerated by `make a11y-report`.

## Conformance target

WCAG 2.2 Level AA across the Flutter citizen app and the Next.js admin console.

## Controls

| WCAG SC | Control |
|---|---|
| 1.3.1 Info and relationships | Semantics widgets + `Semantics` labels on every interactive element |
| 1.4.3 Contrast (minimum) | All palettes audited; contrast ratio ≥ 4.5:1 for text |
| 1.4.11 Non-text contrast | Icons and focus rings ≥ 3:1 |
| 2.1.1 Keyboard | Full keyboard nav; no mouse-only flows |
| 2.4.7 Focus visible | Custom focus ring theme; thickness ≥ 2px |
| 2.5.5 Target size (enhanced AAA, aimed for) | Tap targets ≥ 44×44 dp |
| 3.1.2 Language of parts | Mother-tongue switcher persists selection |
| 4.1.3 Status messages | Toasts via `LiveRegion` equivalent |

## Multimodal & inclusion-first

- Voice input + output for non-literate users — Chirp TTS in English, Hindi, Tamil, Bengali, Telugu, Marathi.
- Kiosk mode for shared village devices (simplified flow, no keyboard chrome).
- Offline-first via Gemini Nano 4 (AICore Dev Preview April 2026) — the audit still runs on 2G.
- Font scaling: supports platform Dynamic Type 80% → 200%.

## Testing

- axe-core in the admin console CI; 0 violations gate.
- `flutter_test` + `integration_test` with `SemanticsTester` on critical flows.
- Manual audit with TalkBack and VoiceOver on P0 flows before each release.
